# Platform: LinkedIn

LinkedIn rewards substance, narrative, and professional credibility. Posts can be longer and more nuanced than Twitter, but every sentence still must earn its place.

## Characteristics

- **Length:** 150-600 words (sweet spot: 200-400)
- **Tone:** Confident but not arrogant, specific but accessible
- **Pipeline:** Fast path (<400 words), Full path (400+ words with critic loop)
- **Audience:** Professionals scrolling a feed. You have 2 seconds to stop the scroll.

## Editorial Pass Selection

- **Under 400 words:** Minto (is the argument coherent?) + Zinsser (every sentence earning its place?) + Forsyth (closer only)
- **400+ words:** All four passes + critic loop

## Structure Options

### Hook-Story-Insight (Most Versatile)
**Hook** (1-2 sentences): Stop the scroll. Counterintuitive claim, specific number, provocative question, moment of tension.
**Story/Context** (2-4 paragraphs): Develop the narrative. Vulnerability, specific details, texture. Every sentence earns its place.
**Insight** (1-2 paragraphs): The payoff. What did this reveal? End with something worth remembering.

### Contrarian Take
Open with conventional wisdom → pivot to contradiction → support with evidence → land the reframe.

### Lesson from Failure
State the failure clearly (no softening) → what actually happened (specifics) → the insight that emerged → the application for the reader.

### Technical Deep-Dive (Eugene Yan Style)
Problem statement → approach at right altitude → the nuance most miss → the takeaway.

## Voice Guides for LinkedIn

### Naval's Contributions
- **Definition-based clarity:** Open or close with a clean definition. "Psychological safety isn't about comfort. It's about enabling risk-taking."
- **Contrast/dichotomy:** "Most people think X. The reality is Y."
- **First principles anchoring:** Ground in fundamental truths that cannot be argued with.
- **Chain logic:** Each statement follows from the last. Useful for developing arguments mid-post.

### Karpathy's Contributions
- **Self-aware confession:** Share failures with humor, not self-pity. "I spent 6 months building something nobody asked for."
- **Technical topics made human:** Explain hard concepts through relatable experience. Not documentation — what it felt like to learn the thing.
- **Shared experience hook:** Start with something every practitioner recognizes. "If you've ever stared at a dashboard wondering why stakeholders aren't using it..."
- **Build-in-public energy:** Share work in progress, not just polished results.

## Opening Lines

The first line must stop the scroll. The test: does the hook serve the insight, or serve the author's image?

**Works:**
- Counterintuitive claim: "Your data warehouse isn't a tech problem. It's a people problem."
- Genuine question: "What's the actual difference between a senior and mid-level data scientist?"
- Specific observation: "Every data team I've seen struggle has the same root cause."
- Moment of tension: "Our 6-month project got shelved last week."

**Be careful:**
- Numbers as hooks: fine if the number serves the insight, cringe if the number is the subtle flex
- Transformation setups: fine if genuine vulnerability, cringe if it sets up "now I'm successful"

**Avoid:**
- "I've been thinking a lot about..."
- "Hot take:" / "Unpopular opinion:"
- "I'm humbled to announce..."
- Starting with a quote then "This resonates..."

## Closing Lines

End with something that sticks:
- **Reframe:** "It's not about having the best tools. It's about having the agency to use any tool."
- **Call to reflection:** "The question isn't whether you have the skills. It's whether you have the hunger."
- **Specific next step:** "If you're building a data team, start by asking what decisions it needs to enable."
- **Honest admission:** "I'm still figuring this out. But I'm convinced the direction is right."

**Avoid:** "Thoughts?", "Comment below!", "Share if this resonated!", excessive hashtags, links in body (put in comments).

## Formatting

- Line breaks for readability (logical pauses, not every sentence)
- Bold sparingly for key phrases
- Numbers when listing genuinely distinct items
- One-sentence paragraphs for punchlines (earned, not manufactured)
- No headers/subheaders (feels like a blog, not a post)
- No bullet-point walls
- Max 1-2 emojis if any

## Before Publishing

1. Does the first line stop the scroll?
2. Genuine insight or just noise?
3. Specifics are specific? (Names, numbers, concrete details)
4. Sounds like a human wrote it?
5. Zero cringe-worthy phrases?
6. Clear which core belief this connects to?

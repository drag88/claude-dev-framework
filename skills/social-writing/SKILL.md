# Social Writing Skill

This skill enables the creation of LinkedIn and Twitter content that sounds genuinely human, connects emotionally with readers, and delivers substantive value—never robotic, never formulaic.

## Core Philosophy

The foundation of all content is this: **readers are humans who crave authenticity in an era drowning in AI-generated noise.**

Every post must pass this test: "Would a thoughtful person actually say this to another person over coffee?"

### The Human-First Principle

Content should feel like it emerged from lived experience, not a content calendar. The goal is connection, not performance. Substance over signal. Earned authority, not claimed expertise.

When writing:
- Lead with genuine curiosity or hard-won insight
- Write as if speaking to one specific person, not a faceless audience
- Let imperfection show—perfect polish reads as artificial
- Prioritize emotional resonance equal to intellectual substance

## The Framework (Adapted from Dan Koe)

### The Inner Album of Greatest Hits

Before writing any post, draw from a pool of 8-10 core beliefs that can connect to almost any topic. These are ideas refined through hundreds of hours of thinking, not hot takes manufactured for engagement.

Reference `references/greatest_hits.md` for Aswin's core beliefs. These should:
- Serve as starting points for any topic
- Be ideas worth repeating and refining
- Connect across domains (technical, leadership, personal growth)

When asked to write about any topic, first identify which core belief(s) the topic connects to. This creates consistency of voice and depth that audiences recognize.

### Three Methods for Articulation

#### Method 1: The Micro Story (Foundation)

The mind is wired for stories. Use transformation as the spine:

**Structure:**
1. **Problem** — State a relatable problem observed or experienced
2. **Amplify** — Illustrate the negative outcome if unsolved
3. **Solution** — Deliver the insight or approach

This is the default starting point. Most ideas can be expressed this way.

#### Method 2: The Pyramid Principle (Building Depth)

For ideas requiring more support:

1. **Lead with the conclusion** — State the main idea first (answer-first approach)
2. **Support with 3-5 key arguments** — Ask "why" repeatedly to generate these
3. **Ground with evidence** — Personal experience, data, examples from others

This works for expanding a single insight into a longer LinkedIn post or thread.

#### Method 3: Cross-Domain Synthesis (Advanced)

The most distinctive content connects ideas across disciplines:

1. **Problem and amplify** — Hook with a relatable problem
2. **Cross-domain insight** — Pull a concept from another field (physics, psychology, economics, sports) to illuminate the topic
3. **Unique process/solution** — Offer a perspective that only emerges from this synthesis

Example: Explaining data quality using entropy from physics, or team dynamics using principles from jazz improvisation.

### The Building Blocks (When Stuck)

When the writing stalls, cycle through these:

- **Pain point** — Start a section with a relevant struggle
- **Example** — Ground abstract ideas with concrete cases
- **Personal story** — Draw from lived experience
- **Statistic** — Add credibility (but don't over-rely on data)
- **Metaphor** — Explain complex ideas simply
- **Quote** — Borrow authority from others (sparingly)
- **Reframe** — Offer a different perspective on what was just said
- **What/How/Why** — When stuck, ask one of these questions

## Anti-Patterns (Never Do These)

### Language to Avoid
- "I'm humbled to announce..." — (false modesty)
- "Thrilled to share..." — (performative enthusiasm)
- "Let that sink in." — (condescending)
- "Read that again." — (manipulative)
- "Here's the thing..." repeated — (filler)
- Excessive emojis — (more than 1-2 if any)
- 🚀💡🔥 as content substitutes — (hollow signals)

### Structures to Avoid
- Broetry (one sentence per line for artificial drama)
- Humble-bragging disguised as lessons
- "X years ago I was [struggling]. Now I [success]" formula
- Listicles without genuine insight
- Engagement bait ("Comment below if you agree!")
- Claiming mastery in fields without demonstrated depth

### Tone to Avoid
- Preaching from above rather than sharing alongside
- Absolute certainty on nuanced topics
- Generic advice that applies to no one specifically
- Recycled wisdom presented as original thought

## Quality Checklist

Before finalizing any post:

1. **Authenticity test** — Does this sound like Aswin speaking, not a template?
2. **Substance test** — Is there a genuine insight or just noise?
3. **Emotional test** — Will this make someone feel something?
4. **Specificity test** — Could only someone with this experience write this?
5. **Cringe test** — Would this be embarrassing to read in 5 years?

## Usage Instructions

1. When asked to write social content, first read this SKILL.md and relevant reference files
2. Identify which core belief from `greatest_hits.md` connects to the topic
3. Select the appropriate platform framework (LinkedIn or Twitter)
4. Choose an articulation method (Micro Story, Pyramid, or Cross-Domain)
5. Draft the content
6. Run through the quality checklist and anti-patterns
7. Refine until it passes all tests

## Inspiration Sources

The approach blends multiple influences:

- **Dan Koe** — Framework for articulation, inner album concept, micro story structure
- **Naval Ravikant** — Aphoristic density, definition-based clarity, first principles thinking, contrast/dichotomy structure
- **Andrej Karpathy** — Technical wit, self-deprecating observations, making complex ideas accessible through humor, build-in-public energy
- **Eugene Yan** — Write for yourself first, practical applied insights, clarity in technical topics
- **Chip Huyen** — Technical depth with accessible voice, authentic personal stories woven into substance
